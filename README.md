# Автоматизация покупки алмазов в MLBB

Бот для автоматической покупки алмазов в Mobile Legends: Bang Bang через [Selectel Mobile Farm](https://selectel.ru/services/cloud/mobile-testing/).

Программа самостоятельно запускает игру, переходит на экран пополнения, выбирает пакет **50 алмазов ($0.99)** и завершает оплату через Google Play — без участия человека.

---

## Как это работает

Автоматизация запускается на реальном Android-устройстве в Selectel Mobile Farm. Подключение — через ADB по TCP. Навигация по игровому интерфейсу построена на **компьютерном зрении (CV)**.

### Конвейер компьютерного зрения

Бот не опирается на фиксированные координаты. Каждое навигационное решение подкреплено одним из двух CV-движков.

#### 1. Template Matching (OpenCV)

```
скриншот → grayscale → multi-scale matchTemplate → NMS → (cx, cy, уверенность)
```

`TemplateMatcher` использует `cv2.matchTemplate` с метрикой `TM_CCOEFF_NORMED` на 7 масштабах (0.7×–1.3×) — это позволяет находить элементы интерфейса вне зависимости от небольших различий в разрешении устройств. Дубликаты обнаружений удаляются методом **Non-Maximum Suppression** (порог IoU > 0.4).

Каждый шаблон — это PNG-вырезка конкретного элемента интерфейса:

| Шаблон | Что ищет |
|---|---|
| `shop_icon.png` | Кнопка "+" рядом со счётчиком алмазов |
| `buy_button.png` | Синяя кнопка "Купить" в Google Play |
| `close_button.png` | Белый × для закрытия окон |
| `x_button.png` | Золотой × на рекламных баннерах |
| `dialog_ok.png` | Кнопка "Принимаю" |
| `google_pay_logo.png` | Шапка листа Google Play Billing |
| `main_menu_bg.png` | Нижняя навигационная панель (детект главного меню) |
| `mlbb_loading_logo.png` | Логотип загрузочного экрана |

#### 2. OCR (EasyOCR)

```
регион скриншота → grayscale → CLAHE → шумоподавление → EasyOCR → OcrResult(текст, уверенность, cx, cy)
```

`OcrEngine` запускает EasyOCR в режиме русский + английский (покрывает весь игровой и системный интерфейс). Перед распознаванием изображение проходит через конвейер предобработки:

1. **Перевод в grayscale** — убирает цветовой шум
2. **CLAHE** (`clipLimit=2.0`, `tileGridSize=8×8`) — улучшает контраст тёмного игрового UI
3. **Шумоподавление (Non-local Means)** — устраняет артефакты сжатия скриншотов

OCR применяется там, где template matching ненадёжен — например, для поиска ценника `"0,99 $"` среди 8 пакетов алмазов, чтобы гарантированно выбрать самый дешёвый, или для нахождения слова `"Купить"` на листе Google Play.

#### Как они работают вместе

Template matching срабатывает первым — он быстрый и детерминированный. OCR используется как fallback и на экранах, где важнее текстовое содержимое, а не визуальный вид. Фоновый поток `Watchdog` применяет оба метода: перед каждой проверкой он вычисляет MD5-хэш скриншота и пропускает OCR, если экран не изменился.

### Сценарий оплаты

```
Запуск MLBB
    └─ ожидание главного меню (шаблоны: mlbb_loading_logo → main_menu_bg)
        └─ тап по полоске алмазов "+" (хардкод: 89.5%, 3.7% экрана)
            └─ экран Пополнения
                └─ OCR ищет "0,99" в левой половине сетки пакетов
                    └─ тап → открывается Google Play (экран переходит в портрет)
                        └─ OCR ищет "Купить"
                            └─ тап → чёрный экран = оплата прошла → SUCCESS
```

### Watchdog

Фоновый поток-демон опрашивает экран каждые 2 секунды. Он использует OCR для обнаружения и закрытия:
- Системных диалогов Android ("Приложение не отвечает")
- Диалога обновления MLBB ("Заметка: версия устарела") — нажимает "Отменить"
- Рекламных баннеров (шаблоны: `close_button`, `x_button`)

Два защитных барьера предотвращают случайные тапы:
- **Главное меню** — если на экране обнаружены сигналы главного меню (`"герои"`, `"магазин"` и др.) и нет диалога обновления, все тапы watchdog'а подавляются
- **Экран пополнения** — если обнаружено слово `"пополнение"`, подавляется и OCR, и template matching

## Установка

### Требования

- Python 3.10+
- [Appium](https://appium.io) с драйвером `uiautomator2`
- Android SDK / `adb` (`brew install android-platform-tools`)
- Аккаунт Selectel Mobile Farm с забронированным Android-устройством

### Установка зависимостей

```bash
cd python-automation
python -m venv .venv
source .venv/bin/activate
pip install -e .
```

### Конфигурация

```bash
cp config.example.yaml config.yaml
# Заполни config.yaml своими данными Selectel и Google-аккаунта
```

### Запуск

```bash
# 1. Подключить ADB (порт берётся из панели Selectel при бронировании)
adb connect adb.mobfarm.selectel.ru:<PORT>

# 2. Запустить Appium (в отдельном терминале)
ANDROID_HOME=~/android-sdk appium

# 3. Запустить автоматизацию
ANDROID_HOME=~/android-sdk ANDROID_SDK_ROOT=~/android-sdk \
  python -m mlbb_automation run --config config.yaml --step payment
```

## Важные детали

- `config.yaml` исключён из репозитория — никогда не коммить реальные данные
- Все скриншоты после старта Appium-сессии делаются через ADB (`adb exec-out screencap`), чтобы обойти Knox-защиту на устройствах Samsung
- Ориентация экрана используется как сигнал состояния: landscape (2408×1080) = сетка пополнения, portrait (1080×2408) = лист оплаты Google Play
